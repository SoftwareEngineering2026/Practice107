

# Мониторинг приложения для генерации QR-кодов

## Описание проекта
Микросервисное приложение для генерации QR-кодов с централизованным сбором логов и метрик.

## Архитектура

### Основные сервисы:
- **Nginx** - внешняя точка входа (порт 8081)
- **Frontend** - пользовательский интерфейс (HTML)
- **Backend** - API для обработки запросов (Flask)
- **Worker** - генерация QR-кодов (длительная задача)

### Мониторинг:
- **Prometheus** - сбор и хранение метрик (порт 9090)
- **Grafana** - визуализация метрик (порт 3000)


## Метрики

### Прикладная метрика
- `qr_requests_total` - общее число запросов на генерацию QR-кода
  - Тип: Counter
  - Увеличивается при каждом POST-запросе к `/generate`

### Системные метрики (собираются автоматически)
- `http_requests_total` - все HTTP-запросы к backend
- `qr_errors_total` - число ошибок при генерации
